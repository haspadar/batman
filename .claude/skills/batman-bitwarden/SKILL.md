# Скил `batman-bitwarden`

Синхронизация серверов Hetzner в Bitwarden (организация Acrobats, коллекция Hetzner).

Источник правды — файл от ronald'а (формат: `пароль hostname ip`).

## Определение сценария

### Сценарий А — Синхронизация из файла
Признаки: пользователь передал путь к файлу или вставил текст в формате `пароль hostname ip` (несколько строк).

→ Запусти `python3 /Users/haspadar/Projects/batman/scripts/sync-bitwarden.py` с переданным файлом или сохрани текст во временный файл и передай его.

Сначала покажи dry-run (`--dry-run`), затем спроси подтверждение, затем запусти без флага.

### Сценарий Б — Правка одной записи
Признаки: пользователь назвал конкретный сервер и/или указал что изменить (hostname, ip, пароль) или сказал удалить.

Действия:
- **Добавить**: создать запись в коллекции Acrobats/Hetzner. Обязательные поля: hostname, ip, пароль — если что-то не указано, запросить у пользователя перед созданием.
- **Обновить**: найти запись по hostname, изменить нужные поля
- **Удалить**: найти запись по hostname, удалить после подтверждения

Используй `bw` CLI напрямую. Перед изменением покажи что именно будет сделано и попроси подтверждение.

### Сценарий В — Ничего не передано
Спроси: **"Что синхронизировать? Передай файл или укажи сервер для правки."**
