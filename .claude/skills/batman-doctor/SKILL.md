# Скил `batman-doctor`

Аудит SSH-доступа к серверам из Bitwarden (организация Acrobats, коллекция Hetzner).

## Шаг 0 — Проверить конфиг-файлы

Перед началом работы проверь наличие и формат файлов. Все пути относительно корня репозитория batman (`/Users/haspadar/Projects/batman`).

### `.bw_password`

```bash
cat /Users/haspadar/Projects/batman/.bw_password
```

Ожидается: непустая строка (пароль Bitwarden).

Если файл отсутствует или пуст — сообщи: `✗ .bw_password не найден или пуст` и предложи создать: `echo "твой_пароль" > .bw_password`

### `.ssh_user`

```bash
cat /Users/haspadar/Projects/batman/.ssh_user
```

Ожидается: одна строка в формате `логин пароль /path/to/key.pub` (ровно 3 части).

Дополнительно проверь, что файл публичного ключа существует:
```bash
# Извлечь путь к ключу (3-е поле) и проверить
KEY_PATH=$(awk '{print $3}' /Users/haspadar/Projects/batman/.ssh_user)
ls "$KEY_PATH"
```

Возможные ошибки:
- `✗ .ssh_user не найден` — предложи создать: `echo "логин пароль ~/.ssh/id_rsa.pub" > .ssh_user`
- `✗ .ssh_user неверный формат` — нужно ровно 3 поля через пробел
- `✗ публичный ключ не найден: <путь>` — приватный ключ тоже отсутствует, создай автоматически:

```bash
# KEY_PATH — путь из .ssh_user без .pub (приватный ключ)
ssh-keygen -t rsa -b 4096 -f "$KEY_PATH" -N "" -C "<логин>@batman"
```

После создания выведи:
```
✓ ключ создан: <путь> (публичный: <путь>.pub)
```

Если все проверки прошли — выведи:
```
✓ .bw_password — OK
✓ .ssh_user — OK (логин: <логин>, ключ: <путь>)
```

И переходи к следующему шагу.

## Шаг 1 — Получить список серверов из Bitwarden

Разблокировка:
```
bw unlock --passwordfile /Users/haspadar/Projects/batman/.bw_password --raw
```

Синк и получение записей из коллекции:
```
bw sync
bw list items --collectionid a879e149-9510-4fbe-9644-b41300e6a521
```

Для каждой записи взять:
- `name` → hostname
- `login.uris[0].uri` → ip
- `login.password` → пароль

Выдели записи с неполными данными (нет ip или нет пароля) — они не будут проверяться.

## Шаг 2 — Проверка SSH

Сформируй входные данные в формате `пароль hostname ip` и передай в скрипт:

```
echo "<данные>" | python3 /Users/haspadar/Projects/batman/scripts/check-ssh.py
```

## Шаг 3 — Итоговый отчёт

Выведи таблицу и резюме:
```
✓ OK           autode-master   116.203.102.213
✗ AUTH_FAIL    danon           159.69.159.91
...

Итого: 17/18 доступны
```

## Шаг 3б — Проверка пользователя

После SSH-проверки, для серверов со статусом OK, передай те же данные в скрипт:

```
echo "<данные>" | python3 /Users/haspadar/Projects/batman/scripts/check-user.py
```

Выведи таблицу рядом с SSH-результатами:
```
✓ OK           autode-master   116.203.102.213   user: ✓ OK
✓ OK           chilo-master    5.161.201.61      user: ✗ MISSING
```

Если есть серверы где юзер не настроен — предложи запустить `/batman-user`.

## Шаг 3в — Проверка входа по ключу

Для серверов где пользователь существует (статус OK или NO_SUDO/NO_KEY из шага 3б), проверь вход по ключу:

```bash
echo "<hostname> <ip>" | python3 /Users/haspadar/Projects/batman/scripts/check-key.py
```

Передай строки в формате `hostname ip` (по одной на каждый сервер).

Результат выведи рядом:
```
✓ OK           autode-master   116.203.102.213
✗ NO_KEY       danon           159.69.159.91
```

Если есть серверы с `NO_KEY` — автоматически запусти `/batman-user` для них.

## Шаг 4 — Проверка /etc/hosts

Используй список серверов из Bitwarden (шаг 1) — те же ip и hostname.

Прочитай текущий блок в `/etc/hosts` между маркерами `# batman begin` и `# batman end`.

Сравни построчно: что есть в Bitwarden vs что прописано в `/etc/hosts`.

```bash
grep -A 1000 '# batman begin' /etc/hosts | grep -B 1000 '# batman end'
```

Выведи расхождения:
```
✗ отсутствует в /etc/hosts: new-server   1.2.3.4
✗ устарел в /etc/hosts:     old-server   5.6.7.8  (в Bitwarden уже нет)
~ актуально: autode-master  116.203.102.213
```

Если расхождений нет — выведи `✓ /etc/hosts актуален` и переходи к шагу 5.

Если есть расхождения — автоматически запусти `/batman-sync-hosts` без запроса пользователя.

## Шаг 5 — Реакция на недоступные серверы

Если есть серверы со статусом не OK — для каждого спроси пользователя: **"Что делать с <hostname> (<статус>)? Удалить или обновить данные?"**

После ответа пользователя — сам вызови `/batman-bitwarden`, передав hostname и нужное действие. Не жди пока пользователь сам запустит скил.

Варианты для каждого проблемного сервера:
- **Удалить** — удалить запись из Bitwarden
- **Обновить пароль** — запросить новый пароль и обновить
- **Обновить IP** — запросить новый IP и обновить
- **Обновить hostname** — запросить новое имя и обновить
